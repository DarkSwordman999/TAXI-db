# TAXI DB

Учебный проект реляционной базы данных для службы такси. Проект покрывает полный цикл работы с PostgreSQL: проектирование таблиц, загрузку данных, аналитические запросы, представления, подзапросы, пользовательские функции, Python-отчеты, графики и графический интерфейс.

## Назначение

База данных хранит сведения о:

- автомобилях службы такси;
- водителях;
- клиентах;
- тарифах;
- поездках;
- сезонности спроса.

Проект подходит для демонстрации лабораторных работ по базам данных и как основа для курсовой работы по теме автоматизации учета поездок службы такси.

## Технологии

- PostgreSQL 16;
- SQL и PL/pgSQL;
- Bash;
- Python 3;
- psycopg2;
- matplotlib;
- Tkinter;
- Git.

## Быстрый запуск

1. Создать базу данных:

```bash
createdb taxi_lab5
```

2. Полностью пересобрать структуру и данные:

```bash
psql -d taxi_lab5 -f setup/rebuild_database.sql
```

3. Проверить основные команды:

```bash
./h 04
./h task1
./h 200
./h 204
```

4. Запустить Python-отчеты:

```bash
python3 -m pip install -r requirements.txt
./PYTHON all
```

5. Запустить графический интерфейс:

```bash
./run_app.sh
```

Если база уже создана и нужно только поправить последовательности `SERIAL` после ручной загрузки данных, можно выполнить:

```bash
psql -d taxi_lab5 -f setup/sync_sequences.sql
```

## Структура базы данных

| Таблица | Назначение |
|---|---|
| `АВТОМОБИЛИ` | автомобили, государственные номера, класс и характеристики |
| `ВОДИТЕЛИ` | ФИО, телефон, стаж, категория прав и рейтинг |
| `КЛИЕНТЫ` | ФИО, телефон, email и дата регистрации |
| `ТАРИФЫ` | стоимость километра, часа и минимальная стоимость по классам |
| `ПОЕЗДКИ` | факты поездок со ссылками на автомобиль, водителя и клиента |
| `СЕЗОН` | соответствие месяца сезону для аналитики |

Связи:

- `ПОЕЗДКИ.автомобиль` -> `АВТОМОБИЛИ.код`;
- `ПОЕЗДКИ.водитель` -> `ВОДИТЕЛИ.код`;
- `ПОЕЗДКИ.клиент` -> `КЛИЕНТЫ.код`.

Для ускорения аналитики добавлены индексы по датам поездок, внешним ключам, классу автомобиля, рейтингу водителя и телефону клиента.

```bash
psql -d taxi_lab5 -At -c "SELECT 'АВТОМОБИЛИ', count(*) FROM АВТОМОБИЛИ UNION ALL SELECT 'ВОДИТЕЛИ', count(*) FROM ВОДИТЕЛИ UNION ALL SELECT 'КЛИЕНТЫ', count(*) FROM КЛИЕНТЫ UNION ALL SELECT 'ПОЕЗДКИ', count(*) FROM ПОЕЗДКИ UNION ALL SELECT 'ТАРИФЫ', count(*) FROM ТАРИФЫ UNION ALL SELECT 'СЕЗОН', count(*) FROM СЕЗОН;"
```

## Python-отчеты

Запускатель Python-части: `./PYTHON`.

| Команда | Описание |
|---|---|
| `./PYTHON 1 [водитель] [класс]` | отчет по водителям и классам |
| `./PYTHON 1s [водитель] [класс]` | отчет со звездочкой внизу |
| `./PYTHON 2 [класс]` | сводная таблица |

| `./PYTHON all` | выполнить все отчеты |

Готовые графики сохраняются в `docs/`.

## Графический интерфейс

GUI находится в `app/gui.py` и запускается командой:

```bash
./run_app.sh
```

Возможности GUI:

- просмотр и фильтрация таблиц;
- статистика по базе;
- запуск SQL-скриптов проекта;
- запуск команд `./h` и `./PYTHON`;
- добавление, редактирование и удаление записей;
- отчеты за период, по дням недели и по часам;
- графики matplotlib;
- экспорт результатов в CSV;
- резервная копия базы через `pg_dump`;
- триграммный поиск через расширение `pg_trgm`.

## Примечания

- В проекте используются русскоязычные имена таблиц и полей, поэтому терминал и редактор должны работать в UTF-8.
- Скрипт `setup/rebuild_database.sql` пересоздает таблицы и данные. Его не стоит запускать на базе, где нужно сохранить ручные изменения.
- Для курсовой работы рекомендуется фиксировать фактические счетчики таблиц после окончательной загрузки данных.
